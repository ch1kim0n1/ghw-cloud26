# Data Schema Definitions

## 1. Purpose

Define the executable MVP SQLite schema and the data rules behind it.

## 2. MVP Database Rules

- database: SQLite
- migration style: SQL files under `backend/scripts/migrations/`
- IDs: text UUID-like strings generated by the application
- local file paths are stored intentionally in MVP
- Azure Blob references may be stored only as temporary artifact metadata
- no ORM required for MVP

## 3. Tables

### products

```sql
CREATE TABLE products (
  id TEXT PRIMARY KEY,
  name TEXT NOT NULL,
  description TEXT,
  category TEXT,
  context_keywords_json TEXT,
  source_url TEXT,
  image_path TEXT,
  created_at TEXT NOT NULL,
  updated_at TEXT NOT NULL
);
```

### campaigns

```sql
CREATE TABLE campaigns (
  id TEXT PRIMARY KEY,
  product_id TEXT NOT NULL,
  name TEXT NOT NULL,
  video_filename TEXT NOT NULL,
  video_path TEXT NOT NULL,
  source_fps REAL NOT NULL,
  duration_seconds REAL NOT NULL,
  target_ad_duration_seconds INTEGER NOT NULL DEFAULT 6,
  created_at TEXT NOT NULL,
  updated_at TEXT NOT NULL,
  FOREIGN KEY (product_id) REFERENCES products(id)
);
```

### jobs

```sql
CREATE TABLE jobs (
  id TEXT PRIMARY KEY,
  campaign_id TEXT NOT NULL,
  status TEXT NOT NULL CHECK (
    status IN ('queued', 'analyzing', 'generating', 'stitching', 'completed', 'failed')
  ),
  current_stage TEXT,
  progress_percent INTEGER NOT NULL DEFAULT 0,
  selected_slot_id TEXT,
  repick_count INTEGER NOT NULL DEFAULT 0,
  error_code TEXT,
  error_message TEXT,
  metadata_json TEXT,
  created_at TEXT NOT NULL,
  started_at TEXT,
  completed_at TEXT,
  FOREIGN KEY (campaign_id) REFERENCES campaigns(id)
);
```

### scenes

```sql
CREATE TABLE scenes (
  id TEXT PRIMARY KEY,
  job_id TEXT NOT NULL,
  scene_number INTEGER NOT NULL,
  start_frame INTEGER NOT NULL,
  end_frame INTEGER NOT NULL,
  start_seconds REAL NOT NULL,
  end_seconds REAL NOT NULL,
  motion_score REAL,
  stability_score REAL,
  dialogue_activity_score REAL,
  longest_quiet_window_seconds REAL,
  narrative_summary TEXT,
  context_keywords_json TEXT,
  action_intensity_score REAL,
  abrupt_cut_risk REAL,
  created_at TEXT NOT NULL,
  metadata_json TEXT,
  FOREIGN KEY (job_id) REFERENCES jobs(id)
);
```

### slots

```sql
CREATE TABLE slots (
  id TEXT PRIMARY KEY,
  job_id TEXT NOT NULL,
  scene_id TEXT NOT NULL,
  rank INTEGER NOT NULL,
  anchor_start_frame INTEGER NOT NULL,
  anchor_end_frame INTEGER NOT NULL,
  quiet_window_seconds REAL NOT NULL,
  score REAL NOT NULL,
  context_relevance_score REAL,
  narrative_fit_score REAL,
  anchor_continuity_score REAL,
  reasoning TEXT NOT NULL,
  status TEXT NOT NULL CHECK (
    status IN ('proposed', 'selected', 'rejected', 'generating', 'generated', 'failed')
  ),
  suggested_product_line TEXT,
  final_product_line TEXT,
  product_line_mode TEXT CHECK (
    product_line_mode IN ('auto', 'operator', 'disabled')
  ),
  rejection_note TEXT,
  generated_clip_path TEXT,
  generated_audio_path TEXT,
  generation_error TEXT,
  metadata_json TEXT,
  created_at TEXT NOT NULL,
  updated_at TEXT NOT NULL,
  FOREIGN KEY (job_id) REFERENCES jobs(id),
  FOREIGN KEY (scene_id) REFERENCES scenes(id),
  UNIQUE (job_id, rank)
);
```

### job_previews

```sql
CREATE TABLE job_previews (
  id TEXT PRIMARY KEY,
  job_id TEXT NOT NULL UNIQUE,
  slot_id TEXT NOT NULL,
  status TEXT NOT NULL CHECK (
    status IN ('pending', 'stitching', 'completed', 'failed')
  ),
  output_video_path TEXT,
  duration_seconds REAL,
  render_retry_count INTEGER NOT NULL DEFAULT 0,
  artifact_manifest_json TEXT,
  render_metrics_json TEXT,
  error_code TEXT,
  error_message TEXT,
  created_at TEXT NOT NULL,
  completed_at TEXT,
  FOREIGN KEY (job_id) REFERENCES jobs(id),
  FOREIGN KEY (slot_id) REFERENCES slots(id)
);
```

### job_logs

```sql
CREATE TABLE job_logs (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  job_id TEXT NOT NULL,
  timestamp TEXT NOT NULL,
  event_type TEXT NOT NULL,
  stage_name TEXT,
  message TEXT NOT NULL,
  details_json TEXT,
  FOREIGN KEY (job_id) REFERENCES jobs(id)
);
```

## 4. Indexes

```sql
CREATE INDEX idx_campaigns_product_id ON campaigns(product_id);
CREATE INDEX idx_jobs_campaign_id ON jobs(campaign_id);
CREATE INDEX idx_jobs_status ON jobs(status);
CREATE INDEX idx_scenes_job_id ON scenes(job_id);
CREATE INDEX idx_slots_job_id ON slots(job_id);
CREATE INDEX idx_slots_job_rank ON slots(job_id, rank);
CREATE INDEX idx_slots_status ON slots(status);
CREATE INDEX idx_job_previews_job_id ON job_previews(job_id);
CREATE INDEX idx_job_logs_job_id_timestamp ON job_logs(job_id, timestamp DESC);
```

## 5. Entity Meanings

### products

Reusable advertised items. A product may come from manual entry, an uploaded image, or a retailer URL reference.

### campaigns

One source video plus one linked product plus campaign-specific duration settings.

### jobs

One asynchronous execution of analysis, slot review, generation, and preview rendering for a campaign.

### scenes

The analyzed scene list returned by the cloud analysis stage.

### slots

Ranked candidate anchor-frame pairs within scenes, including suggested and final product line data.

### job_previews

The one downloadable preview output for the job. Re-rendering the same job updates the existing preview record and increments render retry count when needed.

### job_logs

Human-readable operational audit trail for the dashboard and debugging.

## 7. Data Rules

- frame values always refer to the uploaded source video frame numbering
- slot anchor frames must use the source FPS, not an assumed fixed FPS
- job status remains coarse even when current_stage, slot status, and preview status are more detailed
- generation failure is terminal for the job in MVP
- render failure preserves generated artifacts for retry
- preview output is singular per job in MVP
- provider request IDs belong in internal metadata only, not standard API responses
